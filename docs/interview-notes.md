# 使用者訪談紀錄

> 這份文件由 Codex 依照 `release-packs/01-interview-kit`，用三個 persona sub-agent 模擬評估目前 `/v1/`。這不是正式使用者訪談結論，仍需要小組刪改、補充與確認。

## 我的 v1 prototype 現況

- `/` 保留 Phase 0 起始畫面，`/v1/` 進入「觀測收集與可靠性整理工作台」。
- v1 仍只使用 `src/fixtures/phase-0/messy-reports.json`，把 Phase 0 原始資訊轉成待人工確認的觀測卡。
- 使用者可以新增觀測，記錄「我看到/聽到/知道了什麼」、「我怎麼知道的」、時間、地點或範圍、回報者角色、主觀類型與主觀急迫感。
- 畫面會搜尋疑似相關回報，但只作為比對線索，不會自動合併。
- 使用者可以補充觀測；補充後狀態仍是「有人補充觀測，尚未人工確認」，沒有 confirmed / verified 狀態。
- 新增與補充只存在前端狀態，沒有後端、資料庫、真實送出或外部查核。

## Persona 1：回報者

### 我想完成的事

- 我想快速留下我看到、聽到或轉述的內容，就算地點、時間、類型還不完整。
- 如果我只是幫別人轉述，我希望系統保留這件事，不要讓後面的人以為我親眼確認過。

### 我看不懂的地方

- 「原始查核狀態」有時顯示 `needs_review`、`unverified`，我不確定差別是什麼。
- 「現場補充數」讓我有點不確定：人比較多補充，會不會被別人看成比較可信？

### 我最怕的錯誤

- 我按下「送出新觀測」後，以為已經真的送到某個救災系統，但其實只是前端暫存。
- 我選了「非常急迫」，後面的人把它當成正式優先順序，而不是我的主觀感受。

### 我不敢直接相信的資訊

- 來源寫「官方公告」但內容又說不知道是不是官方公告，這種我不敢直接相信。
- 「有人說」「群組說」「截圖」「家屬來電」這些我都會覺得可能有時間差或轉述錯誤。

### 我希望系統提醒我的事

- 如果我選「聽說或他人告知」「截圖或轉傳」，請提醒我補上是誰說的、什麼時候看到的。
- 如果我沒填地點或時間，希望系統提醒我可以先送出，但後續一定需要人工確認。

### 我想反問開發者的一個問題

- 我送出的新觀測或補充，最後到底會被誰看到？我需要知道它是不是只是練習畫面。

## Persona 2：資訊整理者

### 我想完成的事

- 我想快速看出哪些觀測保留了原文、來源方式、角色、時間與限制。
- 我想知道哪些資訊只是有人補充，哪些仍然完全缺資料，避免交給下一位時看起來太乾淨。

### 我看不懂的地方

- 「原始查核狀態」和「待人工確認 / 原始資訊未查核」同時出現時，我不確定哪個才是我現在該依賴的狀態。
- 「現場補充數」看起來像可靠性訊號，但我不確定它只是數量，還是代表比較可信。

### 我最怕的錯誤

- 有人看到「親眼看到」或「我在現場」就以為這筆已經可以直接行動。
- 沒填時間時如果系統自動帶入現在，我會怕後面的人誤會那是實際觀測時間。

### 我不敢直接相信的資訊

- Phase 0 轉進來的原始資訊，因為角色、取得方式、地點範圍多半還是待辨識。
- 「疑似相關回報」我只能當作比對線索，不敢直接當成同一件事。

### 我希望系統提醒我的事

- 提醒我 `sourceType` 只是資訊怎麼進來，不是可信度。
- 提醒我補充觀測、多人補充、現場補充都不是人工查核完成。

### 我想反問開發者的一個問題

- 當一筆觀測同時有原始未查核、補充未查核、主觀急迫感時，你希望我先看哪一個欄位來判斷「下一步要確認什麼」？

## Persona 3：行動者

### 我想完成的事

- 我想快速知道這筆回報現在能不能拿來行動，還是只能先記下來等人確認。
- 我想看出地點、時間、需求、來源方式哪裡不完整，避免跑錯地方或做錯事。

### 我看不懂的地方

- 「非常急迫」和「不能直接行動」同時出現時，我會不知道自己到底該不該立刻反應。
- 「現場補充數」看起來像是可靠度增加，但我不確定它只是數量，還是真的有人確認過。

### 我最怕的錯誤

- 我看到「現場回報」或「官方公告」就以為可以相信，但其實畫面仍說需要人工確認。
- 有人補充後，我誤以為這筆資訊已經比較可靠，結果補充內容本身也還沒查核。

### 我不敢直接相信的資訊

- Phase 0 原始資訊轉進來的內容，因為角色、取得方式、地點範圍大多還不清楚。
- 「親眼看到」「親耳聽到」這類選項，我會覺得比較可信，但它仍只是回報者自己填的，不等於已查核。

### 我希望系統提醒我的事

- 每次我看到急迫、現場、補充數增加時，都提醒我這些不是確認結果。
- 如果資訊可能過期、地點不完整、或只是轉述，希望不要讓我把它當成可出發任務。

### 我想反問開發者的一個問題

- 這個 v1 畫面是希望行動者也來看，還是只希望我看完後知道「不能直接行動，應交給資訊整理者確認」？

## 人類補充

- 我採用的 AI 回饋：三個 sub-agent 都指出 v1 有保留不確定性，但「現場補充數」「親眼看到」「我在現場」「非常急迫」仍可能被誤讀成可信度或行動優先順序。
- 我不採用的 AI 回饋：不把行動者想知道「能不能行動」直接轉成產品功能，因為這會接近真實救災行動判斷。
- 我自己補充的判斷：目前 v1 主要服務回報者，但資訊整理者與行動者的誤讀風險需要在文件或下一階段流程中保留。
- 我仍不確定的地方：使用者是否會理解「只存在前端狀態」代表沒有真實送出；這需要真人測試或課堂討論確認。
